# Handover — World Cup 2026 Predictor

Status snapshot for whoever picks this up next. The app is feature-complete
for v1 and working end-to-end with real data. Nothing is deployed/hosted
yet — that's the main thing left.

## TL;DR — what this app does

"Your formula. Your World Cup champion." Users adjust weights across **11
factors** (Elo, FIFA ranking, squad quality, league strength, betting odds,
tournament experience, home-continent advantage, GDP, population, travel
burden, heat adaptation), hit Simulate, and get:
- A power-rankings table (all 48 teams, Power Score + Champion %, with
  hover-tooltip explanations, collapsible to top 12).
- A representative group stage (fixtures + standings for all 12 groups).
- A knockout bracket (R32 -> R16 -> QF -> SF -> Final) with scores and
  penalty shootouts where relevant.

## Current state (all done, working)

- **Backend** (`backend/`, FastAPI): `app/main.py` exposes
  `GET /api/factors`, `GET /api/teams`, `POST /api/simulate`. Serves the
  real **48-team, 11-factor** dataset from `backend/data/teams.json` +
  `factors.json` (already built and committed — no pipeline run needed to
  run the app).
- **Data pipeline** (`backend/data_pipeline/`): every `fetch_*.py` is
  implemented and pulls real data (see table below). `build_team_table.py`
  joins everything, min-max normalizes each factor to 0-1, writes
  `backend/data/teams.json` + `factors.json`.
- **Frontend** (`frontend/`, Vite + React + TS): full UI — factor sliders
  with enable/disable checkboxes, per-factor "Solo" buttons, "AI
  Recommended defaults" button (sorted defaults, styled with an AI badge),
  "Randomize" button, and per-factor info-dot that shows the factor
  description on hover or click (tap-to-pin for touch). Results panel has
  the combined power-rankings table (collapsible), group stage, and
  bracket. Defaults to mock data (`VITE_USE_MOCKS=true` via
  `frontend/.env.local`, gitignored) but works against the live backend
  with `VITE_USE_MOCKS=false`.
- **Mocks** (`mocks/`): 16-team "mini World Cup" fixtures generated by
  `mocks/generate_mocks.py`, mirrored into `frontend/src/mocks/`. Keep
  these in sync with `backend/data_pipeline/build_team_table.py`'s
  `FACTORS` list (same ids/order/descriptions) — they're meant to be
  structurally identical, just smaller.
- **Tests**: `backend/tests/test_simulate.py`, 3 tests, all passing
  (includes the Round-of-32 "never repeat a group-stage pairing" bracket
  seeding check).

### The 11 factors (id | category | default weight, sums to 115)

```
elo                  | form    | 20
fifa_rank            | ranking | 15
squad_quality        | squad   | 15
league_strength      | squad   | 10
betting_odds         | market  | 15
tournament_experience| history | 10
home_advantage       | fun     | 5
gdp                  | fun     | 10
population           | fun     | 5
travel_distance      | fun     | 5
heat_adaptation      | fun     | 5
```

Each has a long, hand-written, mutually-distinguishing description (shown
in the frontend's per-factor info tooltip) — see
`backend/data_pipeline/build_team_table.py` for the canonical text, mirrored
in `mocks/generate_mocks.py`.

### Data sources (all real, no synthetic data)

| Factor(s) | Source |
|---|---|
| `elo` | eloratings.net (scraped) |
| `fifa_rank` | Wikipedia "Top 20" (~19/48 teams) + linear-fit estimate from Elo for the rest (`fifa_rank_source` in `raw` says which) |
| `tournament_experience` | hand-curated titles/appearances/best-finish table |
| `gdp`, `population` | World Bank API (`NY.GDP.MKTP.CD`, `SP.POP.TOTL`), batched |
| `squad_quality`, `league_strength` | Kaggle `justdhia/ea-sports-fc-26-player-ratings` (top-26 players/nationality) joined with clubelo per-country top-division Elo |
| `betting_odds` | the-odds-api.com `soccer_fifa_world_cup_winner` outrights |
| `home_advantage` | flat 1.0 for CONCACAF (2026 host confederation), else 0.0 |
| `travel_distance` ("Travel Burden") | hand-curated from a real 2026 schedule/venue analysis (mikami3345.cloudfree.jp), inverted+normalized (shorter = higher score) |
| `heat_adaptation` | World Bank CCKP climate API (June/July avg temps per country) for each squad's club-league countries, vs. average June/July temp across the 16 2026 host cities; inverted+normalized |

## Running locally

Backend:
```bash
cd backend
source .venv/bin/activate   # venv already exists with deps installed
uvicorn app.main:app --reload --port 8000
```
> Gotcha: don't combine `nohup ... & disown` with a following `sleep`/`curl`
> in the *same* shell call — it reliably exits 144 in this sandbox. Start
> the server in one call, curl it in a separate call.

Frontend:
```bash
cd frontend
npm run dev   # http://localhost:5173, defaults to mocks
```

Regenerating real data (only needed if you change a `fetch_*.py` or add a
factor):
```bash
cd backend
source .venv/bin/activate
python -m data_pipeline.build_team_table   # writes data/teams.json, data/factors.json
```
Then regenerate mocks and re-sync to the frontend:
```bash
cd /home/ubuntu/predictor
python3 mocks/generate_mocks.py
cp mocks/teams.json mocks/factors.json mocks/simulate_response.json frontend/src/mocks/
```

### Secrets needed for the data pipeline (not needed to just run the app)

- `backend/.env` (gitignored): `ODDS_API_KEY=...` for `fetch_odds.py`
  (the-odds-api.com).
- `~/.kaggle/kaggle.json` for `fetch_squad_ratings.py` (Kaggle API, needed
  to download `ea_fc26_players.csv`).

Neither is committed; `backend/data/teams.json`/`factors.json` (the
*output* of running these) **is** committed, so the app runs fine without
either secret.
