# API Contract — World Cup Prediction Simulator (v1)

This is the source of truth for the shape of data exchanged between the
frontend and backend. Both tracks (frontend + backend/data) should build
against this document. Sample JSON matching this contract lives in
[`mocks/`](mocks/) (generated by `mocks/generate_mocks.py`) and can be used
by the frontend before the real API exists.

The mocks use a **16-team / 4-group "mini World Cup"** (so the bracket has
3 rounds: Quarterfinals -> Semifinals -> Final). The real v1 backend will
serve all **48 teams / 12 groups**, with a 5-round bracket (Round of 32 ->
Round of 16 -> Quarterfinals -> Semifinals -> Final). The *shapes* are
identical — only array lengths and the `rounds` list differ. Frontend code
should never hardcode "16 teams" or "3 rounds"; always render from the
arrays.

## Endpoints

### `GET /api/factors`

Returns the list of adjustable factors, in display order. Used to render
the sliders.

```json
[
  {
    "id": "elo",
    "label": "Elo Strength",
    "description": "Overall team strength based on historical match results (World Football Elo Ratings).",
    "category": "form",
    "default_weight": 20
  }
]
```

- `id`: stable key, used in the `weights` map sent to `/api/simulate`.
- `default_weight`: 0-100, sums to 100 across all factors by default (not
  enforced — see normalization note below).
- `category`: for grouping sliders in the UI (`form`, `ranking`, `squad`,
  `market`, `history`, `fun`).

See [`mocks/factors.json`](mocks/factors.json) for the full v1 list (11
factors).

### `GET /api/teams`

Returns all teams with their pre-computed, normalized factor scores (0-1)
plus the raw underlying stat for display/tooltips.

```json
{
  "id": "BRA",
  "name": "Brazil",
  "flag": "🇧🇷",
  "confederation": "CONMEBOL",
  "group": "B",
  "factors": { "elo": 0.785, "fifa_rank": 0.826, "...": "..." },
  "raw": { "fifa_rank": 5, "elo_rating": 2089, "...": "..." }
}
```

`factors.*` values are always 0-1 (min-max normalized across the team
pool) and are what the simulation actually multiplies by the user's
weights. `raw.*` is only for human-readable display (e.g. "FIFA Rank #5",
"Elo 2089").

See [`mocks/teams.json`](mocks/teams.json).

### `POST /api/simulate`

Request body — a map of factor id to weight (0-100, doesn't need to sum to
100; the backend normalizes by dividing by the sum):

```json
{
  "weights": {
    "elo": 20,
    "fifa_rank": 15,
    "squad_quality": 15,
    "league_strength": 10,
    "betting_odds": 15,
    "tournament_experience": 10,
    "home_advantage": 5,
    "gdp": 10,
    "population": 5,
    "travel_distance": 5,
    "heat_adaptation": 5
  }
}
```

Response:

```json
{
  "weights_used": { "elo": 0.2, "fifa_rank": 0.15, "...": "normalized so all weights sum to 1" },
  "team_scores": [
    { "id": "ARG", "name": "Argentina", "power_score": 0.7617, "rank": 1 }
  ],
  "champion_probabilities": [
    { "id": "ARG", "name": "Argentina", "probability": 0.345 }
  ],
  "group_stage": [
    {
      "group": "A",
      "matches": [
        { "matchday": 1, "home": "USA", "away": "MEX", "home_goals": 2, "away_goals": 1 },
        { "matchday": 1, "home": "JPN", "away": "CRO", "home_goals": 0, "away_goals": 0 },
        { "matchday": 2, "home": "USA", "away": "JPN", "home_goals": 1, "away_goals": 1 },
        { "matchday": 2, "home": "CRO", "away": "MEX", "home_goals": 3, "away_goals": 0 },
        { "matchday": 3, "home": "USA", "away": "CRO", "home_goals": 1, "away_goals": 2 },
        { "matchday": 3, "home": "MEX", "away": "JPN", "home_goals": 1, "away_goals": 2 }
      ],
      "standings": [
        {
          "id": "USA", "name": "United States",
          "played": 3, "won": 1, "drawn": 1, "lost": 1,
          "goals_for": 4, "goals_against": 4, "goal_difference": 0,
          "points": 4, "qualified": true
        }
      ]
    }
  ],
  "bracket": {
    "rounds": [
      {
        "name": "Quarterfinals",
        "matches": [
          {
            "home": "ARG", "away": "BRA",
            "home_goals": 1, "away_goals": 1,
            "winner": "BRA",
            "penalties": { "home": 3, "away": 4 }
          }
        ]
      }
    ],
    "champion": "ESP"
  },
  "explanation": "Argentina tops your bracket (34.5% of simulations), driven mainly by ..."
}
```

Notes:

- `team_scores`: every team, sorted by `power_score` desc, with `rank`.
  Used for a "your power rankings" table.
- `champion_probabilities`: only teams with non-zero probability across the
  Monte Carlo trials, sorted desc. Used for the headline bar chart.
- `group_stage`: per group, for a single representative simulation (the
  *first* Monte Carlo trial — i.e. "your predicted bracket"):
  - `matches`: every group-stage fixture (6 per group of 4 — round-robin),
    each with a `matchday` (1-3) and a final score (`home_goals`/
    `away_goals`). Render these as the group's fixture list.
  - `standings`: full table with `played/won/drawn/lost/goals_for/
    goals_against/goal_difference/points`, sorted by points then goal
    difference then goals for (the same tiebreakers used to compute
    `qualified`). `qualified` marks the teams that advance (top 2 per
    group, plus the best third-placed teams filling the bracket out to a
    power of two).
- `bracket.rounds`: ordered list of knockout rounds for that same
  representative simulation. Render generically — map over `rounds`, each
  with a `name` and `matches` array of `{home, away, home_goals,
  away_goals, winner, penalties}`. `penalties` is `null` unless the match
  was drawn after full time, in which case it's `{ "home": N, "away": N }`
  (the shootout score) and `winner` reflects the shootout result.
- `explanation`: a short, rule-based natural-language blurb (v1 — no LLM).
  Safe to render as plain text.

## Conventions

- All team references use the 3-letter id (`"BRA"`, `"ARG"`, ...), matching
  `teams[].id`. Frontend should look up display name/flag from the
  `/api/teams` response by id.
- Normalized factor and probability values are floats in `[0, 1]`.
- No auth, no pagination in v1. CORS open to `*` for local dev.
